# Kybernetes v0.1.1

Date: 2026-07-17

Kybernetes v0.1.1 hardens information release without changing the v0.1
runtime-adaptive control model.

## Highlights

- Tool permission no longer implies permission to release unbounded context.
- `permission_boundary` now requires a destination, purpose, minimum payload,
  exclusions, handling decision, approval evidence, residual-risk record, and
  fail-closed fallback when information leaves the current boundary.
- Pressure scenario 069 covers unbounded information release, while scenario
  024 protects lightweight local work from added approval ceremony.
- Repository guidance and validation now match the five pressure-backed skills
  shipped in v0.1, with `kybernetes:loop-governor` remaining the sole canonical
  lifecycle controller.

## Compatibility

- No new skill, port, lifecycle state, runtime adapter, canonical artifact, or
  persistence layer was added.
- Existing `control.md`, `verification.md`, and `manifest.md` records remain
  compatible.
- Local, reversible, low-variety work remains foreground and lightweight.

## Upgrade

Reinstall or update the Kybernetes skills from the repository. Existing durable
run records require no migration.
