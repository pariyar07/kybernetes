# Kybernetes v0.1.0-rc.1

Date: 2026-07-17

First release candidate for the runtime-adaptive harness.

## Included

- Canonical lifecycle independent of native runtime state.
- Lean manifest-driven governor kernel and capability negotiation.
- Dynamic loop architecture, portable workgraph, activation and notification
  bindings, independent verification, learning capture, and loop closeout.
- Five installable `kybernetes:*` skills backed by pressure scenarios 001-068.

## Portable Guarantees

- Recoverable waits never map to irreversible native terminal blocked.
- One parent writer owns canonical state; children submit revisioned evidence.
- Detached work requires notification or accepted manual checkpoints.
- Completion requires independent, current, rejection-capable evidence.

## Runtime-Dependent Behavior

Native goal editing, scheduling, events, notifications, workers, hooks, and
retirement depend on the callable surface observed at runtime. Documentation is
not treated as proof of agent-callable capability.

## Known Limitations

- File leases are cooperative, not transactional distributed locks.
- Live runtime surfaces vary by provider, plan, app, CLI, and session.
- Transactional orchestration is deferred until pressure evidence requires it.
