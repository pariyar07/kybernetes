# v0.1.2 Release Verification

Date: 2026-07-17

## Deterministic Evidence

- All `tests/validate-*.rb` validators pass, including the new acceptance-boundary
  validator.
- `actionlint`, workflow YAML parsing, skill-frontmatter checks, repository
  hygiene checks, and runtime-neutrality checks pass.
- Pressure scenarios 070-073 cover passing evidence without acceptance,
  dependent-system changes without an accountable owner, brownfield implicit
  constraints, and learning promotion without owner approval.
- Scenario 024 preserves local, reversible work without a separate acceptance
  ceremony.
- The governor kernel remains below its lean-loading threshold at 162 lines.

## Boundary Evidence

- `control.md` remains the only current-truth record and holds conditional owner
  verdicts.
- `verification.md` remains evidence truth; no `acceptance.md` or second
  canonical artifact exists.
- Missing acceptance uses existing recoverable `waiting_human`.
- `manifest.md`, `reference-index.md`, runtime adapter shape, skill count, and
  lifecycle state vocabulary are unchanged.
- Workgraph admission distinguishes implementation ownership from accountable
  acceptance ownership before dependent-system effects.
- Verification and closeout report pending acceptance without issuing the owner
  verdict themselves.

## Limitations

- Kybernetes records and governs owner decisions; it cannot prove that an actor
  has organizational authority beyond the evidence available to the active run.
- Brownfield constraint discovery remains proportionate and evidence-dependent;
  unavailable historical or operational constraints must be recorded as gaps.
