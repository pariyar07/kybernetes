# Kybernetes v0.1.4

Date: 2026-07-23

Kybernetes v0.1.4 adds enforceable liveness and runtime-portable trajectory
control for durable adaptive work.

## Highlights

- Safe runtime execution and consistent state can no longer substitute for
  outcome progress. A strategy that reaches its cumulative no-progress cap must
  adapt, pause, escalate, supersede, or retire.
- Finite goals and continuing programs now have distinct contracts. Continuing
  cycle health never becomes finite completion evidence.
- Progress observations, safety decisions, completion verification, and
  trajectory decisions remain separate sensor questions.
- Strategy state is keyed by `strategy_id` across activations, reconstruction,
  workers, chats, and schedules.
- Readiness tests approved actionable capacity and fallback coverage rather than
  actuator existence alone.
- ChatGPT Work mode and Claude Cowork receive distinct complete L3 bindings,
  alongside Codex, Claude Code, and portable-core.
- Every detached, scheduled, or fresh activation probes its own required
  callable capabilities and fails closed to a portable fallback when necessary.
- Pressure scenarios 075-080 and structural validators cover stalled detached
  runs, typed funnel semantics, reachability, authority, maintenance/event waits,
  and runtime capability assumptions.

## Compatibility

- The five-skill inventory and sole canonical governor are unchanged.
- No canonical lifecycle state, manifest schema revision, or installable skill
  name was added or removed.
- Existing clear, reversible foreground work remains on the lean kernel without
  mandatory trajectory machinery.
- Durable high/extreme recurring or detached runs may add strategy, progress
  window, continuing-program, and trajectory fields to `control.md`.
- `trajectory.md` remains conditional and never replaces `control.md` or
  completion-focused `verification.md`.

## Upgrade

Install or update all five Kybernetes skills from the exact `v0.1.4` tag after
the release tag is published from the merged `main` commit. Do not install this
behavior from an unmerged branch or an untagged repository snapshot.

Verification details: [v0.1.4-verification.md](v0.1.4-verification.md).
