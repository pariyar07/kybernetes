# Kybernetes v0.1.3

Date: 2026-07-17

Kybernetes v0.1.3 makes its historical skill naming truthful without changing
the installed control surface.

## Highlights

- `$parallel-coordinator` is documented as the retired v0 name, not an
  installed compatibility alias.
- Users are directed to install and invoke `$kybernetes:loop-governor`.
- Pressure scenario 074 covers invocation of the retired name.
- Repository validation now rejects a renewed unsupported alias claim or an
  accidentally installable parallel-coordinator skill.
- The possible future `kybernetes:parallel-coordinator` specialist remains
  separately pressure-gated in architecture planning.

## Compatibility

- No working alias was removed; the package did not expose one.
- The five installable skills and sole canonical lifecycle governor are
  unchanged.
- Runtime bindings, lifecycle states, `control.md`, `verification.md`, and
  manifest routing are unchanged.

## Upgrade

Reinstall or update the Kybernetes skills from the repository. Replace any
historical `$parallel-coordinator` invocation with
`$kybernetes:loop-governor`.

Verification details: [v0.1.3-verification.md](v0.1.3-verification.md).
