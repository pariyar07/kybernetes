# Portable Launcher Example

```text
Use kybernetes:loop-governor.
Load and follow .kybernetes/<slug>/control.md before significant work.

Objective: <one concrete outcome>.
Done when: <verifiable acceptance criteria>.
Verify with: <commands or evidence>.
Constraints: <limits and approval boundaries>.
Execution profile: generate task type, role stance, risk posture, artifacts, verification style, communication cadence, and HITL triggers.
Loop readiness: define setpoint, sensor/evidence, actuators, state, stop condition, and HITL boundary.
Altitude: choose stay, down, up, stack, or stop at checkpoints.

If this runtime lacks goals or subagents, use the control record as the durable objective and continue in the main thread unless the user explicitly approves another execution mode.
```

If a runtime cannot invoke namespaced skills, still record `kybernetes:loop-governor`
as the canonical skill in the control record and use the equivalent local skill
folder.
