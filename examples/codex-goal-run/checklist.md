# Checklist Example

- [ ] Sense task variety.
- [ ] Create or read the durable control record.
- [ ] Check loop readiness: setpoint, sensor/evidence, actuators, state, stop, boundary.
- [ ] Choose loop altitude: stay, down, up, stack, or stop.
- [ ] Ask only missing high-impact questions.
- [ ] Generate execution profile.
- [ ] Bind to runtime.
- [ ] Decide whether worker fan-out creates leverage; record why or why not.
- [ ] Create task contracts for workers if stacking bounded child loops.
- [ ] Integrate worker summaries.
- [ ] Verify DONE with objective evidence.
- [ ] Record impediments, escalations, decisions, and learnings.
- [ ] Record next activation or explicitly close the loop.
